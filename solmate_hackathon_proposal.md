## 1. 목적 및 필요성

### 1-1. 문제 정의

> "죽어서야 발견"… 고독사 급증, 보험·특수청소 시장 키운다 — 한국경제 (2025)

매년 고독사가 늘고 있지만, 복지 시스템은 이들을 찾지 못하고 있다.

| 지표 | 수치 | 출처 |
|------|------|------|
| 전국 복지대상자 | 1,153만 명 | 보건복지부 |
| 읍면동 복지 담당자 | 1만 1천 명 | 행정안전부 |
| 담당자 1인당 관리 인원 | 평균 **1,027명** | 2020, 복지부 |
| 발굴 후 지원 연결률 | **43%** (절반 이상 미연결) | 보건복지부 2023 |
| 위기정보 수집 주기 | 격월 (연 6회) | 복지부 위기정보시스템 |

### 1-2. 왜 사각지대가 생기는가

빅데이터망에 걸리지 않는 계층이 존재한다.

- **주소지 ≠ 실거주지**: 고시원·쪽방·여관 거주자
- **자기 신청 불가**: 정신질환, 인지저하, 사회적 고립
- **이웃 무관심**: 1인 가구 증가로 주변 인지 자체가 없음

> 장례지도사 A씨: "이런 분들은 유가족이 시신 인수를 거부하면 무연고자로 공영장례를 치른다. 이웃의 관심과 지자체의 지원이 절실하다." — 한국경제

### 1-3. 기존 해결 시도의 한계

DB손해보험은 2017년 고독사 관련 보험을 출시했지만 **8년간 계약 1건**에 그쳤다.
이유는 단 하나 — **위험 임차인이 어디 있는지 알 수 없어서**.

**Solmate는 이 데이터를 만드는 플랫폼이다.**

## 2. 서비스 개요

### "국가가 못 찾는 사람을, 이웃이 찾고, AI가 연결하고, 블록체인이 증명한다"

### 2-1. 전체 파이프라인

```
[발굴 — B2C 시민 참여]
이웃 제보 / 가족 케어 등록 / 자가 신청
          ↓
[판단 — AI 플랫폼]
위기도 스코어링 + Soli 챗봇 케어
모바일 신분증으로 신원 확인 (제보자 책임, 대상자 자격)
          ↓
[집행 — B2G 담당자 도구]
행정복지센터 담당자 대시보드
승인 → 지원 집행
          ↓
[기록 — OmniOne Chain]
집행 이력 온체인 투명 기록
```

### 2-2. 핵심 기능

#### 🟦 시민 앱 (B2C)

| 기능 | 설명 |
|------|------|
| 이웃 제보 | 주소 + 상황 입력 → 모바일 신분증으로 제보자 신원 확인 → 허위 신고 방지 |
| 가족 케어 등록 | 지방 부모님을 Soli가 매일 말 걸기 → 응답 패턴 이상 시 가족 + 복지사 알림 |
| 자가 복지 신청 | Soli와 대화 → 받을 수 있는 복지 자동 매칭 → 모바일 신분증으로 자격 확인 → 원스톱 신청 |

#### 🟧 AI 챗봇 Soli

- 24/7 감정 케어 및 일상 대화
- 위기 키워드 감지 → 위기도 스코어 자동 산출
- 단순 챗봇이 아닌 **"신뢰를 쌓는 첫 접점"** — 공무원이 갑자기 문 두드리는 게 아니라 Soli가 먼저 관계를 만듦

#### 🟩 담당자 대시보드 (B2G)

- "오늘 케어 필요한 사람 N명" 자동 우선순위 표시
- 위기 스코어 + 제보 내역 + Soli 대화 요약 통합 제공
- 승인 버튼 하나로 지원 집행 연결

#### 🟪 OmniOne Chain 기록

- 제보 → 판단 → 집행 전 과정 온체인 기록
- 지원금 사용처 투명화
- 기업 CSR 기부 집행 증명

## 3. 기술 스택 및 필수조건 충족

### 3-1. 필수조건 — 모바일 신분증 활용

모바일 신분증(OmniOne CX)이 **세 곳에서 자연스럽게 사용**된다.

| 사용 시점 | 역할 |
|-----------|------|
| 이웃 제보 시 | 제보자 신원 확인 → 허위 신고 방지 |
| 복지 자격 확인 시 | 대상자 본인 확인 → 즉시 자격 검증 |
| 지원 집행 직전 | 수급자 본인 확인 → 부정 수급 방지 |

억지로 끼운 것이 아닌, 서비스의 신뢰 구조 자체가 모바일 신분증에 의존한다.

### 3-2. 선택조건 — Open DID + OmniOne Chain (가점 +10%)

| 기술 | 활용 |
|------|------|
| Open DID | 시민 제보자 DID 발급 → VC 기반 신원 증명 |
| OmniOne Chain | 제보-판단-집행 전 과정 온체인 기록, CSR 기부 투명화 |

### 3-3. 전체 기술 스택

```
Frontend   Next.js + TypeScript (기존 Solmate 코드베이스 활용)
Backend    NestJS + PostgreSQL
AI         Claude API (한국어 복지 상담 최적화)
Auth       OmniOne CX (모바일 신분증), Open DID
Blockchain OmniOne Chain
Infra      Vercel + AWS
```

## 5. 비즈니스 모델

### "사회문제를 해결하면서 지속 가능한 수익을 만든다"

### 5-1. 4개 레이어 수익 구조

#### Layer 1 — B2G 구독 SaaS (안정 수익)

행정복지센터 담당자용 AI 도구 월정액 구독

```
전국 읍면동 약 3,500개
담당자 1인당 월 10만원 기준

파일럿   10개 동  →  월  100만원
확장    100개 동  →  월 1,000만원
전국  3,500개 동  →  월    3.5억
```

정부 SW 조달 등록 → 수의계약 진입 가능

#### Layer 2 — B2B2C "안심 시니어 보험" (대중화 채널)

보험사가 Solmate 기능을 탑재한 보험 상품을 출시, 자녀에게 판매

```
[자녀 — B2C]
부모님 떨어져 사는 자녀가 보험 가입
보험료 납부

        ↓

[보험사 — B2B]
Solmate에 '위기 감지 솔루션 이용료' 매달 지불
보험사 영업망으로 전국 자동 확산

        ↓

[Solmate]
Soli가 부모님께 매일 말 걸기
이상 감지 시 자녀 + 복지사 동시 알림
모바일 신분증으로 자녀-부모 관계 인증 (DID 기반)
```

**왜 보험사가 원하냐**
```
기존 고독사 보험  →  8년간 계약 1건 (리스크 데이터 없어서 못 팔았음)
Solmate 탑재 후  →  실시간 모니터링으로 선제 개입 가능
                 →  보험금 지출 감소 + 상품 경쟁력 확보
                 →  자녀 시장 (3040) 신규 공략 가능
```

#### Layer 3 — B2B CSR 기부 집행 증명 (브랜드 수익)

기업 CSR 예산을 Solmate 플랫폼으로 집행, OmniOne Chain으로 투명성 증명

```
기존 기업 기부  →  "1억 기부했습니다" 한 줄
Solmate 기부   →  "1억이 127명에게 전달됐고
                   식료품 43%, 의료비 31%, 긴급생활비 26%
                   수혜자 평균 위기도 7.2 → 4.1 감소"
                →  ESG 보고서 자동 생성
```

```
수익 구조:
집행 금액의 3~5% 플랫폼 수수료
ESG 리포트 자동 생성 구독료 월 50만원
```

#### Layer 4 — B2B 보험사 데이터 API (스케일 수익)

Solmate가 축적한 사회적 고립 데이터를 익명화·집계 후 보험사에 API 제공

```
Solmate 보유 데이터 (기존 어디에도 없는 데이터):
- 사회적 고립도 스코어
- 이웃 네트워크 밀도
- Soli 대화 감정 패턴
- 위기 제보 수신 이력

→ 익명화·집계 처리 (개인 특정 불가)
→ 보험사 언더라이팅 API 제공
→ 보험사: 고독사 리스크 계산 가능 → 상품 설계 가능
→ Solmate: API 구독 수수료
```

```
국내 손해보험사 31개 중 10개사 계약 시
연간 API 구독료 1억 × 10사 = 연 10억
```

### 5-2. 성장 로드맵

| 단계 | 기간 | 전략 | 수익원 |
|------|------|------|--------|
| 생존 | 2026 | 해커톤 입상 + 창업 지원금 | 정부 지원 |
| 검증 | 2027 | 파일럿 지자체 10개 + 보험사 1개 계약 | B2G + B2B2C |
| 성장 | 2028 | 보험사 확대 + CSR 기업 계약 | B2B2C + B2B |
| 확장 | 2029~ | 일본·대만 고령화 시장 진출 | 글로벌 |

> 일본에서는 고독사 관련 보험·청소 시장이 이미 활성화되어 있음.
> 한국이 모델을 먼저 정립하면 K-복지 플랫폼 수출 가능.
