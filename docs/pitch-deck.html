2. 제안 요약서핵심 요약

복지 위기 신호를 제때 발견하고, 신뢰 가능한 집행 기록으로 남깁니다

문제

복지대상자는 2019년 1,153.1만 명에서 2023년 1,488.5만 명으로 29% 늘었지만, 발굴 후 실제 지원 연결은 약 43.6%에 그친다.

해결 방식

격월 행정 데이터의 빈틈을 Soli AI 상시 감지, 담당자 판단, 온체인 지원 완료 증명으로 연결한다.

모바일 신분증

가입·로그인, 수급자 집행 확인 두 지점에 적용한다.

Open DID / OmniOne Chain

지원 확인 VC 발급·VP 검증과 지원 완료 증명 txId를 MVP 필수 성공 기준으로 구현해 가점 +10%를 확보한다.

개발·시연 범위

OmniOne CX 가입·로그인 → Soli 감지 → 담당자 승인 → 본인 재확인 → VC 발급·VP 검증 → txId 표시까지 구현한다.

지원 연결 보조

복지 매칭은 핵심 감지 이후 필요한 경우 받을 가능성이 있는 복지를 안내하는 보조 기능이다.

왜 지금까지 잘 안 됐나

  • 신청주의 장벽: 도움이 필요한 사람이 직접 신청하고 증빙해야 한다.
  • 수급자 배제 역설: 이미 소액 지원을 받는 순간 더 큰 위기에서 제외될 수 있다.
  • 아동복지 단절 절벽: 만 18세 퇴소 이후 취약한 가정으로 돌아가며 보호가 끊긴다.
  • 과부하된 현장 인력: 응급 관리 요원 1인이 156명을 담당하며 격월 데이터만으로는 상시 위기 감지가 구조적으로 불가능하다.

no-one-alone은 이 지체를 줄이기 위해 상시 안부, 담당자 우선순위화, 신뢰 가능한 집행 기록을 하나의 흐름으로 묶는다.
