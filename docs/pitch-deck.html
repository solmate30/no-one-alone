2. 제안 요약서핵심 요약

복지 위기 신호를 제때 발견하고, 신뢰 가능한 집행 기록으로 남깁니다

문제

복지대상자는 증가하지만 발굴 후 실제 지원 연결률은 약 44%에 머무른다.

해결 방식

Soli AI 선제 감지, 담당자 돌봄 연결, 온체인 지원 완료 증명을 하나의 흐름으로 연결한다.

모바일 신분증

가입·로그인, 수급자 집행 확인 두 지점에 적용한다.

Open DID / OmniOne Chain

지원 확인 VC 발급·VP 검증과 지원 완료 증명 txId를 MVP 필수 성공 기준으로 구현해 가점 +10%를 확보한다.

개발·시연 범위

OmniOne CX 가입·로그인 → Soli 감지 → 담당자 승인 → 본인 재확인 → VC 발급·VP 검증 → txId 표시까지 구현한다.

지원 연결 보조

복지 매칭은 핵심 감지 이후 필요한 경우 받을 가능성이 있는 복지를 안내하는 보조 기능이다.

사각지대가 생기는 이유

  • 주소지와 실거주지가 다른 고시원·쪽방 거주자
  • 정신질환, 인지저하, 사회적 고립으로 자기 신청이 어려운 대상자
  • 이웃이 이상 징후를 알아차려도 명확한 신고 채널 부족

제안의 목적

행정 데이터가 놓치는 위기 신호를 Soli 정기 안부와 대화 패턴으로 포착하고, AI가 담당자 판단에 필요한 근거를 요약해 돌봄 연결 시간을 줄인다.

4. 제안내용의 상세 설명서비스 흐름

발굴, 판단, 집행, 기록을 하나의 업무 흐름으로 연결합니다

01 발굴

Soli 정기 안부

담당자가 등록한 대상자에게 Soli가 안부를 묻고 응답 공백과 위기 키워드를 감지한다.

02 판단 보조

Soli AI 근거 요약

위기 키워드, 응답 공백, 감정 변화를 요약하고 고위험 근거를 표시한다.

03 집행

담당자 승인

담당자가 대시보드에서 우선순위와 근거 요약을 보고 최종 판단한다.

04 기록

OmniOne Chain 증명

VC 발급, 승인, 집행 이벤트의 해시를 기록해 사후 검증 가능성을 확보한다.

AI는 수급 여부나 지원 여부를 자동 결정하지 않는다. 최종 승인과 집행은 담당자가 수행하고, Soli는 판단 근거를 정리하는 보조 도구로 제한한다.

4. 제안내용의 상세 설명신뢰 구조

OmniOne 기술을 복지 집행의 신뢰 레이어로 사용합니다

구분사용 시점역할검증 가치
모바일 신분증가입·로그인, 집행 직전해커톤 제공 SDK·가이드 기반 OmniOne CX 표준 인증창 연동안전한 서비스 진입·부정 수급 방지
Open DID / VC지원 확인 VC 발급, VP 검증DID 해시와 VC 검증 결과로 행위 주체 연결지원 연결 사실의 사후 검증 가능성 확보
OmniOne ChainVC 발급·승인·집행 이벤트개인정보 원문 없이 이벤트 해시 기록다기관 감사와 CSR 집행 증빙 비용 절감

온체인 기록 대상

  • VC 메타데이터 해시 + 발급 타임스탬프
  • 승인자 해시 + 케이스 해시 + 승인 시각
  • 수급자 DID 해시 + 지원 종류·금액 해시 + txId

개인정보 경계

MVP 데모에서는 실제 주소·상세 상황 설명을 저장하지 않는다. OmniOne Chain에는 DID 해시, 이벤트 해시, 지원 완료 증명 해시만 기록한다.
