## 1. 전체 화면 구조

```
no-one-alone
├── 시민 앱 (B2C)
│   ├── 홈 / 온보딩
│   ├── 이웃 제보 플로우
│   ├── 가족 케어 등록 플로우
│   └── 자가 복지 신청 플로우
├── AI 챗봇 Soli
│   └── 대화 화면 (전 플로우 공통 진입점)
└── 담당자 대시보드 (B2G)
    ├── 위기 우선순위 리스트
    ├── 대상자 상세
    └── 지원 집행 승인
```

---

## 2. 이웃 제보 플로우

```
[진입] 홈 > "이웃이 걱정돼요" 버튼
  ↓
[화면 1] 제보 정보 입력
  - 주소 입력 (도로명/지번)
  - 상황 설명 텍스트 (이상 징후 서술)
  - "다음" 버튼
  ↓
[화면 2] 모바일 신분증 인증 (OmniOne CX)
  - "제보자 신원 확인이 필요합니다" 안내
  - OmniOne CX 앱 연동 → DID VC 제출
  - 허위 신고 방지 동의 체크
  ↓
[화면 3] 제보 완료
  - "제보가 접수되었습니다" 확인 메시지
  - 담당자 연결 예상 시간 안내
  - 온체인 기록 TX 해시 표시 (투명성)
```

**모바일 신분증 인증 포인트:** 화면 2 — 제보자 DID 발급·검증

---

## 3. 가족 케어 등록 플로우

```
[진입] 홈 > "부모님 돌봄 등록" 버튼
  ↓
[화면 1] 부모님 정보 입력
  - 부모님 연락처
  - 거주 주소
  - Soli 대화 빈도 설정 (매일 / 격일)
  ↓
[화면 2] 관계 인증 (DID 기반 자녀-부모 관계 확인)
  - 자녀 본인 OmniOne CX 인증
  - 부모님 동의 링크 발송 (SMS)
  - 부모님 OmniOne CX 수락
  ↓
[화면 3] Soli 케어 시작
  - "내일부터 Soli가 부모님께 매일 말을 겁니다" 안내
  - 알림 설정: 응답 없을 시 자녀 알림 활성화
  - 케어 현황 대시보드 미리보기
  ↓
[알림] 이상 감지 시
  - 부모님 응답 패턴 이상 → 자녀 푸시 알림
  - 위기도 스코어 임계값 초과 → 복지사 동시 알림
```

**모바일 신분증 인증 포인트:** 화면 2 — 자녀·부모 DID 관계 인증

---

## 4. 자가 복지 신청 플로우

```
[진입] 홈 > "복지 도움 받기" 버튼 또는 Soli 챗봇
  ↓
[화면 1] Soli 대화
  - "어떤 어려움이 있으신가요?" 첫 질문
  - 대화 흐름에서 상황·필요 자동 파악
  - 위기 키워드 감지 → 위기도 스코어 내부 산출
  ↓
[화면 2] 복지 매칭 결과
  - "받으실 수 있는 복지가 N개 있습니다" 안내
  - 매칭된 복지 카드 리스트 (복지명, 지원 내용, 신청 방법)
  ↓
[화면 3] 자격 확인 (OmniOne CX)
  - 선택한 복지 자격 검증을 위한 본인 확인
  - OmniOne CX VC 제출 → 즉시 자격 검증
  ↓
[화면 4] 원스톱 신청
  - 자격 확인된 복지 목록
  - "한번에 신청하기" 버튼
  - 신청 접수 확인 + 온체인 기록
```

**모바일 신분증 인증 포인트:** 화면 3 — 대상자 본인 확인·자격 검증

---

## 5. 담당자 대시보드 플로우 (B2G)

```
[진입] 로그인 (담당자 계정)
  ↓
[화면 1] 메인 대시보드
  - "오늘 케어 필요한 사람 N명" 배너
  - 위기도 스코어 내림차순 우선순위 리스트
    └── 각 카드: 이름(익명화) / 스코어 / 제보 유무 / 최근 Soli 응답 상태
  ↓
[화면 2] 대상자 상세
  - 위기 스코어 그래프 (최근 30일)
  - 이웃 제보 내역
  - Soli 대화 요약 (AI 요약, 원문 아님)
  - 기존 지원 이력 (온체인 기록 연동)
  ↓
[화면 3] 지원 집행 승인
  - 지원 종류 선택 (긴급생활비 / 식료품 / 의료비 등)
  - 금액 입력
  - "승인" 버튼 → 수급자 OmniOne CX 본인 확인 요청
  ↓
[화면 4] 집행 완료
  - 온체인 기록 TX 해시 생성
  - ESG/CSR 연계 시 자동 보고서 항목 추가
```

**모바일 신분증 인증 포인트:** 화면 3 — 수급자 본인 확인·부정 수급 방지

---

## 6. Soli 챗봇 공통 화면

```
[진입] 어느 플로우에서든 접근 가능한 플로팅 버튼
  ↓
[대화 화면]
  - 채팅 UI (말풍선 형식)
  - Soli 아바타 (친근한 캐릭터)
  - 내부적으로: 위기 키워드 실시간 감지 → 스코어 갱신
  - 24/7 응답 (Claude API)
  ↓
[위기 감지 시 분기]
  - 스코어 임계값 미만: 일반 대화 지속
  - 스코어 임계값 초과 (경미): "전문 복지사 연결 도와드릴까요?" 제안
  - 스코어 임계값 초과 (위급): 담당자 즉시 알림 + 자녀 알림 발송
```
