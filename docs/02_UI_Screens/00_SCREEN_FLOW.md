## 1. 전체 화면 구조

Solmate는 단일 반응형 웹 MVP로 구현한다. 별도 모바일 앱 없이 경로와 권한으로 화면을 분리한다.

```
/                       — 서비스 소개 및 진입
/join                   — 현장 QR 자가 가입 (55세 이상 안부 연결 신청)
/checkin                — Soli 정기 안부 화면 (대상자)
/welfare                — Soli 복지 매칭 화면 (대상자)
/verify                 — OmniOne CX 본인 확인
/officer                — 담당자 대시보드 메인
/officer/targets        — 정기 안부 대상자 등록·관리
/officer/cases/:id      — 위기 케이스 상세 + 집행 승인
/proof/:txId            — 온체인 기록 확인 (공개)
```

---

## 2. 현장 QR 자가 가입 플로우

주민센터·복지관·경로당·보건소·아파트 관리사무소·병원·약국 등 현장 접점에서 QR 스캔 또는 /join 직접 접속으로 시작하는 흐름이다. "취약계층 등록"이 아니라 "55세 이상 안부 연결 신청"으로 낙인 없이 진입한다.

```
[현장 접점] QR 스캔 또는 /join 직접 접속
  ↓
[화면 1] /join — 55세 이상 안부 연결 신청
  - "혼자 지내실 때 담당자가 더 빨리 확인할 수 있는 안전 연결 서비스입니다" 안내
  - 동의 범위 선택
      필수: Soli 정기 안부 대화
      선택: 생활위기 체크리스트 공유
  ↓
[화면 2] /verify — OmniOne CX 본인 확인
  - 모바일 신분증 인증창 호출
  - 검증 완료 → DID 해시 저장, JWT 발급
  ↓
[화면 3] Soli 연결 완료
  - "내일부터 Soli가 안부를 드릴게요" 안내
  - /checkin 화면으로 이동
  ↓
[백그라운드] 담당자 대시보드
  - 신규 가입자 자동 추가 (담당 지역 코드 기반 배정)
  - 이후 Soli 대화 신호가 쌓이면 위기 우선순위 반영
```

**모바일 신분증 인증 포인트:** 화면 2 — 본인 확인·가입 동의

## 3. 담당자 등록 기반 정기 안부 플로우

```
[진입] /officer/targets > "정기 안부 대상 등록" 버튼
  ↓
[화면 1] 대상자 기본 정보 입력
  - 대상자 연락처
  - 관할 지역 코드 또는 행정동 선택 (MVP 데모에서는 실제 주소 원문 미저장)
  - Soli 대화 빈도 설정 (매일 / 격일)
  ↓
[화면 2] 대상자 동의 및 본인 확인
  - 대상자에게 동의 링크 발송 (SMS)
  - /verify → OmniOne CX 본인 확인
  - Soli 정기 안부 동의 저장
  ↓
[화면 3] Soli 정기 안부 시작
  - "내일부터 Soli가 정기 안부를 시작합니다" 안내
  - 담당자 대시보드에 대상자 추가
  - 응답 공백 알림 임계값 설정
  ↓
[알림] 이상 감지 시
  - 응답 공백 또는 위기 키워드 누적
  - 위기도 스코어 임계값 초과 → 담당자 즉시 알림
```

**모바일 신분증 인증 포인트:** 화면 2 — 대상자 본인 확인·정기 안부 동의

---

## 4. Soli 복지 매칭 플로우

```
[진입] /welfare 또는 /checkin 대화 중 복지 필요 감지
  ↓
[화면 1] Soli 대화 (/checkin 또는 /welfare)
  - "어떤 어려움이 있으신가요?" 첫 질문
  - 대화 흐름에서 연령·가구·상황 힌트 파악
  - 기본 조건 입력: 연령대, 거주지역, 가구 형태
  - 선택형 상황 체크리스트: 실직, 질병, 주거 불안, 돌봄 공백, 긴급 생계 등
  - 위기 키워드 감지 → 위기도 스코어 내부 산출
  ↓
[화면 2] 복지 매칭 결과
  - "받으실 수 있는 복지가 N개 있습니다" 안내
  - 대화 맥락 + 기본 조건 + 체크리스트를 결합한 한국사회보장정보원 API 기반 매칭 카드 리스트
    └── 복지명 / 지원 내용 / 신청 방법
  ↓
[화면 3] 신청 연결
  - "신청하러 가기" → 복지로(bokjiro.go.kr) 또는 정부24 외부 링크
```

## 5. 담당자 대시보드 플로우

```
[진입] /officer (담당자 계정 로그인)
  ↓
[화면 1] 메인 대시보드 (/officer)
  - "오늘 케어 필요한 사람 N명" 배너
  - 위기도 스코어 내림차순 우선순위 리스트
    └── 각 카드: 익명화 이름 / 스코어 / 최근 Soli 응답 상태
  ↓
[화면 2] 대상자 상세 (/officer/cases/:id)
  - 위기 스코어 그래프 (최근 30일)
  - Soli 대화 요약 (AI 요약, 원문 아님)
  - 기존 지원 이력 (온체인 기록 연동)
  ↓
[화면 3] 지원 집행 승인
  - 지원 종류 선택 (긴급생활비 / 식료품 / 의료비 등)
  - 금액 입력
  - "승인" 버튼 → /verify OmniOne CX 수급자 본인 확인
  ↓
[화면 4] 집행 완료
  - 온체인 기록 txId 생성
  - /proof/:txId 링크 표시
```

**모바일 신분증 인증 포인트:** 화면 3 — 수급자 본인 확인·부정 수급 방지

---

## 6. Soli 챗봇 공통 화면

```
[진입] /checkin 또는 /welfare
  ↓
[대화 화면]
  - 채팅 UI (말풍선 형식)
  - Soli 아바타 (친근한 캐릭터)
  - 내부적으로: 위기 키워드 실시간 감지 → 스코어 갱신
  - 24/7 응답 (Claude API SSE 스트리밍)
  ↓
[위기 감지 시 분기]
  - 스코어 임계값 미만: 일반 대화 지속
  - 스코어 임계값 초과 (경미): "전문 복지사 연결 도와드릴까요?" 제안
  - 스코어 임계값 초과 (위급): 담당자 즉시 알림
```
