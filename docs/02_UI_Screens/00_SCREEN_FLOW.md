## 1. 전체 화면 구조

NOA는 단일 반응형 웹 MVP로 구현한다. 별도 모바일 앱 없이 경로와 권한으로 화면을 분리한다.

```
/                       — 서비스 소개 및 진입
/join                   — 현장 QR 자가 가입 (55세 이상 안부 연결 신청)
/welfare                — 솔이 복지 매칭 화면 (대상자)
/verify                 — OmniOne CX 본인 확인
/officer                — 담당자 대시보드 메인
/officer/targets        — 정기 전화 대상자 등록·관리
/officer/cases/:id      — 위기 케이스 상세 + 집행 승인
/proof/:txId            — 온체인 기록 확인 (공개)
/checkin                — 솔이 웹 채팅 화면 (SMS 또는 직접 링크로 진입 — 전화 이후 사용자가 선택하는 보조 채널)
```

**4단계 관리 구조**: 1단계는 모든 대상자 공통 AI 전화다. 2단계는 전화 이후 SMS 또는 웹 채팅(/checkin) 중 사용자가 편한 채널을 선택하는 보조 대화 단계로, 두 채널은 병렬 운영된다. 3단계에서 모든 신호가 담당자 대시보드에서 통합 분류되며, 4단계에서 담당자가 직접 개입한다. MVP는 웹(Next.js) 단일 배포이며 네이티브 앱은 해커톤 이후 개발한다.

**접근성 설계 원칙**: 가입 시 OmniOne CX 본인확인은 스마트폰이 1회 필요하다. 가입 이후 일상적인 서비스 이용(솔이 전화 수신·SMS 답장)은 WiFi·인터넷 없이 작동한다. 블록체인 기록·담당자 대시보드는 담당자 측에서만 사용하며 어르신이 직접 접근할 필요가 없다. 저소득 독거노인·취약계층이 가장 도움이 필요하면서 동시에 인터넷 접근성이 가장 낮은 사람임을 설계에 반영한다.

**고령층 현장 인증 보조 시나리오**: OmniOne CX 본인확인은 반드시 대상자 본인이 직접 수행한다. 스마트폰 조작이 익숙하지 않은 고령층의 경우, 주민센터·복지관 담당자가 현장에서 QR 접속과 인증창 진행 단계를 옆에서 안내하는 방식으로 보조한다. 인증 행위 자체는 대상자 본인이 하며, 담당자는 화면 안내 역할로 제한된다. 가입 이후 솔이 전화 수신·SMS 답장은 인터넷 없이 작동하므로, 모바일신분증은 최초 1회 가입과 집행 직전 확인에만 필요하다.

## 2. 현장 QR 자가 가입 플로우

주민센터·복지관·경로당·보건소·아파트 관리사무소·병원·약국 등 현장 접점에서 QR 스캔 또는 /join 직접 접속으로 시작하는 흐름이다. 이 흐름은 수급 자격을 확정하는 복지 신청이 아니라, 솔이가 먼저 전화할 수 있도록 수신 동의를 확보하고 필요 시 복지 연결로 이어지는 안부 연결 신청이다. "취약계층 등록"이 아니라 "55세 이상 안부 연결 신청"으로 낙인 없이 진입한다.

```
[현장 접점] QR 스캔 또는 /join 직접 접속
  ↓
[화면 1] /join — 55세 이상 안부 연결 신청
  - "혼자 지내실 때 담당자가 더 빨리 확인할 수 있는 안전 연결 서비스입니다" 안내
  - 전화번호 입력 (솔이 발신 번호로 사용)
  - 동의 범위 선택
      필수: 솔이 정기 전화 수신
      선택: 생활위기 체크리스트 공유
  ↓
[화면 2] /verify — OmniOne CX 본인 확인
  - 모바일 신분증 인증창 호출
  - 검증 완료 → DID 해시 저장, JWT 발급
  ↓
[화면 3] 솔이 연결 완료
  - "내일부터 솔이가 전화를 드릴게요" 안내
  - 첫 전화 예정 시각 표시
  ↓
[백그라운드] 담당자 대시보드
  - 신규 가입자 자동 추가 (담당 지역 코드 기반 배정)
  - 이후 솔이 통화 신호가 쌓이면 위기 우선순위 반영
```

**모바일 신분증 인증 포인트:** 화면 2 — 본인 확인·가입 동의

## 3. 담당자 등록 기반 솔이 정기 전화 플로우

```
[진입] /officer/targets > "정기 전화 대상 등록" 버튼
  ↓
[화면 1] 대상자 기본 정보 입력
  - 대상자 전화번호 (솔이 발신 대상)
  - 관할 지역 코드 또는 행정동 선택 (MVP 데모에서는 실제 주소 원문 미저장)
  - 솔이 전화 주기 설정 (기본: 주 1회 / 위기 레벨에 따라 자동 조절)
  ↓
[화면 2] 대상자 동의 및 본인 확인
  - 대상자에게 동의 링크 발송 (SMS)
  - /verify → OmniOne CX 본인 확인
  - 솔이 정기 전화 수신 동의 저장
  ↓
[화면 3] 솔이 정기 전화 시작
  - "내일부터 솔이가 정기적으로 전화를 드립니다" 안내
  - 담당자 대시보드에 대상자 추가
  - 부재 시 알림 임계값 설정 (기본: 2회 연속 부재)
  ↓
[백그라운드] 솔이 자동 발신 (Vercel Cron + 알리고 API)
  - 지정 시각에 아웃바운드 전화 발신
  - 통화 종료 후 STT 변환 → Claude 위기 분석 → 담당자 요약 생성
  ↓
[알림] 이상 감지 시
  - 2회 연속 부재 또는 위기 키워드 누적
  - 위기도 스코어 임계값 초과 → 담당자 즉시 알림
```

**모바일 신분증 인증 포인트:** 화면 2 — 대상자 본인 확인·정기 전화 수신 동의

---

## 4. 솔이 복지 매칭 플로우

```
[진입] 솔이 통화 중 복지 필요 감지 또는 /welfare 직접 접속
  ↓
[화면 1] /welfare — 솔이 복지 매칭
  - "어떤 어려움이 있으신가요?" 첫 질문
  - 기본 조건 입력: 연령대, 거주지역, 가구 형태
  - 선택형 상황 체크리스트: 실직, 질병, 주거 불안, 돌봄 공백, 긴급 생계 등
  - 위기 키워드 감지 → 위기도 스코어 내부 산출
  ↓
[화면 2] 복지 매칭 결과
  - "받으실 수 있는 복지가 N개 있습니다" 안내
  - 기본 조건 + 체크리스트를 결합한 한국사회보장정보원 API 기반 매칭 카드 리스트
    └── 복지명 / 지원 내용 / 신청 방법
  ↓
[화면 3] 신청 연결
  - "신청하러 가기" → 복지로(bokjiro.go.kr) 또는 정부24 외부 링크
```

## 5. 담당자 대시보드 플로우

```
[진입] /officer (담당자 계정 로그인)
  ↓
[화면 1] 메인 대시보드 (/officer)
  - "오늘 케어 필요한 사람 N명" 배너
  - 위기도 스코어 내림차순 우선순위 리스트
    └── 각 카드: 익명화 이름 / 스코어 / 최근 솔이 통화 상태 (통화 완료 / 부재 N회)
  ↓
[화면 2] 대상자 상세 (/officer/cases/:id)
  - 위기 스코어 그래프 (최근 30일)
  - 솔이 통화 AI 요약 (AI 요약, 원문 아님) + 부재 이력
  - 기존 지원 이력 (온체인 기록 연동)
  ↓
[화면 3] 지원 집행 승인
  - 지원 종류 선택 (긴급생활비 / 식료품 / 의료비 등)
  - 금액 입력
  - "승인" 버튼 → /verify OmniOne CX 대상자 본인 확인
  ↓
[화면 4] 집행 완료
  - 온체인 기록 txId 생성
  - /proof/:txId 링크 표시
```

**모바일 신분증 인증 포인트:** 화면 3 — 대상자 본인 확인·부정 수급 방지

## 6. 솔이 전화 통화 메인 플로우

솔이의 1단계 채널은 아웃바운드 전화다. 대상자가 앱을 설치하거나 인터넷 없이 전화만 받으면 된다. 전화 이후 2단계에서 SMS 또는 웹 채팅(/checkin) 중 편한 채널을 선택해 소통을 이어간다.

```
[백그라운드] Vercel Cron Jobs — 지정 시각 자동 트리거
  ↓
[발신] 알리고 API → 대상자 전화번호로 아웃바운드 발신
  ↓
[통화 중/통화 후 처리]
  - CLOVA Speech (STT): 음성 → 텍스트 변환
  - GPT-4o mini: 대화 흐름 생성 (자유 안부 → 확인 질문)
  - CLOVA Voice (TTS): 텍스트 → 솔이 목소리 출력
  - 내부적으로: 위기 키워드·감정·목소리 에너지 신호 분석
  ↓
[통화 종료 후] 자동 처리
  - 전체 통화 STT 텍스트 → Claude API 위기 분석
  - 위기도 스코어 갱신
  - 담당자 대시보드 AI 요약 카드 생성
  ↓
[위기 감지 시 분기]
  - 스코어 0–3 (정상): 다음 정기 통화 예약
  - 스코어 4–6 (주의): 통화 주기 상향 조정
  - 스코어 7–8 (경고): 담당자 알림 + 통화 주기 최대
  - 스코어 9–10 (위급): 담당자 즉시 긴급 알림
  ↓
[부재 시 분기]
  - 1회 부재: 2시간 후 재시도
  - 2회 연속 부재: 담당자 알림
  - 3회 연속 부재: 담당자 긴급 알림
```

## 7. 웹 채팅 보조 채널 (/checkin)

전화 이후 사용자가 선택하는 2단계 채널 중 하나다. SMS가 편한 사용자는 문자로, 웹이 익숙한 사용자는 /checkin으로 접속해 솔이와 대화를 이어간다. 두 채널은 병렬 운영되며 모두 NOA 서버가 수신·처리한다.

```
[진입] SMS 안내 링크 클릭 또는 직접 /checkin 접속
  ↓
[대화 화면]
  - 웹 채팅 UI (SMS 안내 링크로 진입 가능)
  - 솔이 아바타
  - 내부적으로: 위기 키워드 감지 → 스코어 반영
  ↓
[위기 감지 시]
  - 전화 메인 플로우와 동일한 분기 처리
```
