> Created: 2026-05-08 01:39

## 1. 설계 원칙

| 원칙 | 설명 |
|:---|:---|
| 신뢰 우선 | 모든 화면에서 "이 데이터는 안전하게 처리됩니다" 인식을 심어줌 |
| 접근성 | 고령층·저학력 사용자를 고려한 큰 글씨, 단순 동선 |
| 단계별 정보 노출 | 한 화면에 하나의 행동 요청 — 과부하 방지 |
| 투명성 시각화 | 온체인 TX 해시를 사용자가 볼 수 있게 표시 |

## 2. 사용자 유형별 UI 원칙

### 2-1. 시민 앱 (B2C)

**타겟**: 이웃 제보자, 3040 자녀, 복지 신청자

| 항목 | 원칙 |
|:---|:---|
| 폰트 크기 | 기본 16px 이상, 제목 20px 이상 |
| 버튼 크기 | 최소 44×44px (터치 타겟) |
| 색상 | 안정감 있는 파랑·초록 계열, 경고는 주황·빨강 |
| 언어 | 쉬운 우리말 — 전문 용어 사용 금지 |
| 진행 상태 | 스텝 인디케이터로 현재 위치 명시 |
| 오류 메시지 | "인증에 실패했습니다" → "신분증 앱을 열고 다시 시도해 주세요" |

**핵심 화면 컴포넌트:**

```
<ReportCard>       — 이웃 제보 접수 카드
<CareStatus>       — 부모님 케어 현황 요약
<WelfareMatch>     — 매칭된 복지 카드 리스트
<OmniOneButton>    — 모바일 신분증 인증 트리거 버튼
<ChainProof>       — 온체인 기록 TX 표시 배지
```

### 2-2. 담당자 대시보드 (B2G)

**타겟**: 읍면동 행정복지센터 사회복지사

| 항목 | 원칙 |
|:---|:---|
| 레이아웃 | 테이블 + 카드 혼합, 한 화면에 최대 정보 밀도 |
| 우선순위 색상 | 위기도 High(빨강) / Medium(주황) / Low(초록) |
| 인터랙션 | 키보드 단축키 지원, 빠른 승인 가능 |
| 데이터 표시 | 익명화된 표시 (이름 마스킹), 클릭 시 상세 |
| 알림 | 실시간 위기 알림 배너 (웹소켓) |

**핵심 화면 컴포넌트:**

```
<PriorityList>      — 위기도 기준 정렬 목록
<CrisisDetail>      — 대상자 위기 상세 (스코어 그래프 + 제보 + Soli 요약)
<ApprovalModal>     — 지원 집행 승인 모달
<ChainHistory>      — 온체인 집행 이력 타임라인
<DashboardMetrics>  — 오늘 케어 완료 N명, 대기 N명 요약
```

## 3. Soli 챗봇 UI 설계

### 3-1. 원칙

- **친근함**: 딱딱한 공공기관 말투 금지 — "안녕하세요, 저는 Soli예요. 오늘 어떠세요?"
- **비위협적**: 위기 감지 시 직접적 질문 대신 간접적 탐색
- **자연스러운 흐름**: 선택지 제공 + 자유 입력 모두 허용
- **신뢰 형성 우선**: 복지 연결 전에 충분한 관계 쌓기

### 3-2. 대화 UI 컴포넌트

```
<ChatBubble>         — 말풍선 (Soli / 사용자 구분)
<QuickReplyBar>      — 빠른 응답 선택지 (2~4개)
<SoliAvatar>         — Soli 캐릭터 아이콘
<TypingIndicator>    — Soli 응답 대기 표시
<CrisisAlert>        — 위기 감지 시 소프트 알림 배너
<WelfareProposal>    — Soli가 제안하는 복지 카드 인라인 표시
```

### 3-3. 위기 감지 UX 분기

| 위기도 수준 | Soli 반응 | UI 변화 |
|:---|:---|:---|
| 정상 (0-3) | 일반 대화 유지 | 변화 없음 |
| 주의 (4-6) | "힘드시겠어요. 도움받을 수 있는 방법을 알아볼까요?" | 복지 제안 카드 표시 |
| 경고 (7-8) | "전문 선생님과 연결해 드릴게요" | 상담 연결 버튼 강조 |
| 위급 (9-10) | "지금 바로 도움을 요청할게요" | 즉시 담당자·가족 알림 발송 |

## 4. OmniOne CX 인증 UX 가이드라인

### 4-1. 인증 진입 원칙

- 인증이 필요한 이유를 먼저 설명한 후 버튼 노출
- 버튼 레이블: "신분증으로 확인하기" (기술 용어 사용 금지)
- 인증 실패 시 재시도 안내 + 대안 경로 제공

### 4-2. 인증 화면 플로우

```
[안내 팝업]
"제보자 신원 확인을 위해 모바일 신분증이 필요합니다.
 허위 신고를 방지하고 소중한 제보를 보호합니다."
  ↓
[버튼] "신분증으로 확인하기"
  ↓
OmniOne CX 앱으로 딥링크 전환
  ↓
[성공] 앱 복귀 → 인증 완료 배지 + 다음 단계
[실패] 오류 메시지 + 재시도 / 고객센터 연결
```

### 4-3. 인증 완료 후 표시

- 인증 완료 배지: "신원 확인됨" (초록 체크)
- DID 해시 일부 표시 (전체 노출 금지)
- 온체인 기록 TX 해시 링크 제공

## 5. 반응형 및 접근성

| 항목 | 기준 |
|:---|:---|
| 모바일 최우선 | 시민 앱: 375px 기준 설계 |
| 데스크탑 | 담당자 대시보드: 1280px 기준 |
| 색상 대비 | WCAG AA 기준 (4.5:1 이상) |
| 키보드 접근 | 담당자 대시보드 전 기능 키보드 조작 가능 |
| 고령층 배려 | 시민 앱 폰트 크기 조절 옵션 |
