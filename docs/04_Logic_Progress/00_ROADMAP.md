## 6. 추후 검토 아이디어 (미확정)

현재 로드맵에 포함되지 않은 아이디어들. 시장 검증 이후 타이밍을 보고 검토한다.

### 복지 마이데이터 플랫폼

**개념**: 금융 마이데이터처럼, 시민이 보건복지부·고용부·지자체 등 여러 기관에 분산된 자신의 복지 수혜 이력과 자격 정보를 DID 지갑에 모아 직접 관리하는 구조.

**현재 no-one-alone과의 차이**: no-one-alone은 위기 감지·발굴이 핵심이고 복지 데이터를 플랫폼이 중개한다. 복지 마이데이터는 데이터 주권을 시민에게 돌려주는 구조로, 정부 정책 연동이 필수 조건이다.

**검토 조건**:
- 보건복지부 "복지 마이데이터" 관련 정책 추진 여부 확인
- 금융 마이데이터 API 표준처럼 복지 분야 API 개방 여부
- no-one-alone 파일럿 운영 후 데이터 주권 수요가 확인되는 시점
