# Product Specs — Solmate MVP
> Created: 2026-05-08
> Last Updated: 2026-05-08

## 1. 개발 단계

**현재 상태**: MVP (배포 완료)
- Next.js + TypeScript 기반 서비스 운영 중
- AI 챗봇 Soli 구현 및 배포 완료
- 한국 공공데이터 API 연동 완료
- OmniOne CX (모바일 신분증) + OmniOne Chain 연동: 해커톤 결선까지 추가 예정

## 2. 기술 스택

| 레이어 | 기술 |
|:---|:---|
| Frontend | Next.js + TypeScript |
| Backend | NestJS + PostgreSQL |
| AI | Claude API (한국어 복지 상담 최적화) |
| Auth | OmniOne CX (모바일 신분증), Open DID |
| Blockchain | OmniOne Chain |
| Infra | Vercel + AWS |

## 3. 핵심 기능 명세

### 3-1. 시민 앱 (B2C)

#### 이웃 제보

| 항목 | 내용 |
|:---|:---|
| 입력 | 주소 + 상황 텍스트 |
| 신원 확인 | OmniOne CX 모바일 신분증 (제보자 본인 확인) |
| 목적 | 허위 신고 방지, 제보자 책임 귀속 |
| 출력 | 담당자 대시보드 위기 알림 생성 |

#### 가족 케어 등록

| 항목 | 내용 |
|:---|:---|
| 대상 | 지방 거주 부모님 (고령층) |
| 방식 | Soli가 매일 말 걸기 |
| 감지 | 응답 패턴 이상 시 → 가족 + 복지사 동시 알림 |
| 인증 | DID 기반 자녀-부모 관계 확인 |

#### 자가 복지 신청

| 항목 | 내용 |
|:---|:---|
| 방식 | Soli와 대화 → 받을 수 있는 복지 자동 매칭 |
| 자격 확인 | OmniOne CX 본인 확인 → 즉시 자격 검증 |
| 신청 | 원스톱 신청 연결 |

### 3-2. AI 챗봇 Soli

| 기능 | 상세 |
|:---|:---|
| 24/7 감정 케어 | 일상 대화 + 정서 지지 |
| 위기 감지 | 위기 키워드 감지 → 위기도 스코어 자동 산출 |
| 복지 매칭 | 대화 맥락 기반 수급 가능 복지 자동 추천 |
| 첫 접점 역할 | 공무원 방문 전 신뢰 관계 형성 |

**위기도 스코어링 기준 (예정)**:
- 대화 감정 패턴 (부정 키워드 빈도, 절망 표현)
- 응답 빈도 이상 (평소 대비 응답 감소)
- 이웃 제보 수신 이력
- 사회적 고립도 지표

### 3-3. 담당자 대시보드 (B2G)

| 기능 | 상세 |
|:---|:---|
| 우선순위 표시 | "오늘 케어 필요한 사람 N명" 자동 정렬 |
| 통합 뷰 | 위기 스코어 + 제보 내역 + Soli 대화 요약 |
| 집행 연결 | 승인 버튼 → 지원 집행 연결 |

### 3-4. OmniOne Chain 기록

| 기록 항목 | 내용 |
|:---|:---|
| 제보 이력 | 제보자 DID + 타임스탬프 + 상황 해시 |
| 판단 이력 | 위기도 스코어 + 담당자 승인 기록 |
| 집행 이력 | 지원 종류 + 금액 + 수혜자 DID |
| CSR 기부 | 기업명 + 집행 금액 + 수혜자 통계 → ESG 리포트 자동 생성 |

## 4. 필수조건 충족 (해커톤 Track 2)

### 모바일 신분증 활용 (필수조건)

| 사용 시점 | 역할 |
|:---|:---|
| 이웃 제보 시 | 제보자 신원 확인 → 허위 신고 방지 |
| 복지 자격 확인 시 | 대상자 본인 확인 → 즉시 자격 검증 |
| 지원 집행 직전 | 수급자 본인 확인 → 부정 수급 방지 |

### Open DID + OmniOne Chain (선택조건 +10%)

| 기술 | 활용 |
|:---|:---|
| Open DID | 시민 제보자 DID 발급 → VC 기반 신원 증명 |
| OmniOne Chain | 제보-판단-집행 전 과정 온체인 기록, CSR 기부 투명화 |

## 5. MVP 범위 (해커톤 데모)

**포함:**
- Soli 챗봇 (위기 키워드 감지 + 위기도 스코어 출력)
- 이웃 제보 플로우 (모바일 신분증 연동 시뮬레이션)
- 담당자 대시보드 (위기 우선순위 리스트)
- OmniOne Chain 집행 기록 시연

**미포함 (이후 단계):**
- 실제 행정 시스템 연동
- 보험사 API 연동
- 재무 실데이터

## 6. Non-Functional Requirements

| 항목 | 요구사항 |
|:---|:---|
| 개인정보 보호 | Soli 대화 데이터 암호화 저장, 담당자만 접근 가능 |
| 온체인 데이터 | 집계·익명화 처리 후 기록 (개인 특정 불가) |
| 가용성 | Soli 24/7 응답 (Claude API 기반) |
| 보안 | 모바일 신분증 연동 시 OmniOne CX 표준 프로토콜 준수 |
