## 1. 문서 목적

이 문서는 Solmate가 실제 현장에서 어떻게 발견되고, 어떻게 신뢰를 얻고, 어떻게 사회취약계층의 안전과 생활 지원으로 연결되는지 정리하는 지속 업데이트 문서다.

제안서의 기술 설명이 "무엇을 만들 수 있는가"를 설명한다면, 이 문서는 "도움이 필요한 사람이 어디서 이 서비스를 만나고, 왜 동의하고, 담당 공무원은 어떤 근거로 움직일 수 있는가"를 설명한다.

핵심 질문은 다음과 같다.

| 질문 | 이 문서에서 다루는 답 |
|:---|:---|
| 누가 대상인가? | 55세 이상 1인·고립 위험 시니어를 우선 대상으로 하되, 낙인을 줄이기 위해 55세 이상 누구나 시작 가능한 안부 연결 창구로 설계한다. |
| 어디서 알게 되는가? | 주민센터, 복지관, 경로당, 보건소, 아파트 관리사무소, 생활지원사 방문, 병원·약국, 공과금 상담 접점에서 만난다. |
| 왜 동의하는가? | "감시"가 아니라 본인이 위험할 때 담당자가 더 빨리 확인할 수 있는 안전망으로 설명한다. |
| 공무원에게 어떤 변화가 생기는가? | 흩어진 민원·전화·방문 기록 대신 오늘 먼저 확인할 고위험 신호를 우선순위로 볼 수 있다. |

## 2. 핵심 메시지

> Solmate는 55세 이상 1인·고립 위험 시니어라면 누구나 시작할 수 있는 동의 기반 디지털 안부 창구다. 주민센터, 복지관, 보건소, 경로당, 고독사 예방 캠페인에서 모바일 신분증으로 본인확인을 하고 Soli와 연결되면, 담당 공무원이 생활위기 신호를 더 빨리 확인할 수 있다.

제안서와 발표에서는 아래 문장을 기준 메시지로 사용한다.

> 도움이 필요한 사람이 서류를 들고 찾아오기 전, Solmate가 먼저 안부를 묻고 담당자에게 신호를 보냅니다.

주의할 표현:

| 피해야 할 표현 | 대체 표현 |
|:---|:---|
| 시니어를 추적한다 | 본인 동의 기반으로 안부와 생활위기 신호를 확인한다 |
| 취약계층 정보를 자동 조회한다 | 사용자가 동의한 범위의 정보와 제출 증빙을 지원 판단에 연결한다 |
| 공무원이 모든 생활 데이터를 본다 | 담당자가 필요한 최소 정보와 위험 신호 요약을 확인한다 |
| 고위험자만 가입한다 | 55세 이상 누구나 가입하고, 이후 신호에 따라 지원 우선순위를 분류한다 |

## 3. 대상 범위와 낙인 완화 전략

### 3-1. 대상 범위

Solmate의 현장 진입 대상은 "이미 위기라고 판정된 사람"에 한정하지 않는다. 실제 현장에서는 본인이 취약계층이라고 인정하기 어렵거나, 지원 대상인지 모른 채 지내는 경우가 많다. 따라서 초기 캠페인은 다음처럼 넓게 잡는다.

| 구분 | 설명 |
|:---|:---|
| 기본 대상 | 55세 이상 시니어 |
| 우선 대상 | 55세 이상 1인가구, 고립 위험군, 만성질환자, 기초연금·긴급복지 상담 경험자 |
| 현장 추천 대상 | 복지 담당자, 보건소 방문건강관리 인력, 생활지원사, 통장·반장이 안부 확인 필요성을 느낀 주민 |
| 본인 신청 대상 | "혼자 살고 있어 불안하다", "요즘 고지서·식사·병원 방문이 어렵다"고 느끼는 주민 |

### 3-2. 낙인을 줄이는 프레이밍

서비스를 "취약계층 등록"으로 설명하면 진입 장벽이 생긴다. 현장에서는 아래처럼 말해야 한다.

| 기존 프레이밍 | Solmate 프레이밍 |
|:---|:---|
| 위기가구 등록 | 55세 이상 안부 연결 신청 |
| 복지 대상자 관리 | 혼자 지내는 분을 위한 디지털 안부 창구 |
| 위험군 추적 | 본인 동의 기반 생활 안전 확인 |
| 긴급 지원 대상 선별 | 도움이 필요할 때 담당자가 빨리 알아차리는 연결망 |

## 4. 현장 접점 설계

### 4-1. 주민센터

주민센터는 가장 강한 공공 신뢰 접점이다.

| 상황 | Solmate 연결 방식 |
|:---|:---|
| 기초연금 상담 | 55세 이상 안부 연결 캠페인 안내 |
| 긴급복지 상담 | Soli 생활위기 체크리스트 등록 |
| 전입신고·1인가구 확인 | "혼자 지내시면 안부 연결을 신청할 수 있습니다" 안내 |
| 복지 담당자 방문 전 | 모바일 신분증 본인확인 후 Soli 정기 안부 시작 |

현장 장면:

> 주민센터 입구에 "55세 이상 1인가구 안부 연결 신청" 안내문이 붙어 있다. 공무원은 "휴대폰 있으세요? 모바일 신분증으로 본인확인만 하면, 불편하실 때 저희가 더 빨리 확인할 수 있습니다"라고 설명한다.

### 4-2. 복지관·경로당

복지관과 경로당은 시니어가 방어감을 낮추고 설명을 들을 수 있는 생활 접점이다.

| 상황 | Solmate 연결 방식 |
|:---|:---|
| 경로당 설명회 | QR 가입 안내, 현장 보조 등록 |
| 복지관 프로그램 | 55세 이상 디지털 안부 캠페인 운영 |
| 스마트폰 교육 | 모바일 신분증 발급·Soli 사용법 함께 안내 |
| 점심·건강 프로그램 | 식사·건강·외출 상태를 자연스럽게 안부 주제로 연결 |

현장 장면:

> 복지사가 "요즘 혼자 지내시다 갑자기 연락이 안 되면 누가 알아차릴까요?"라고 묻는다. 참여자는 QR을 찍고, 모바일 신분증으로 본인확인을 마친 뒤 Soli의 첫 안부 질문을 받는다.

### 4-3. 보건소·방문건강관리

보건소는 질병, 복약, 이동성 저하와 고립 위험이 만나는 접점이다.

| 상황 | Solmate 연결 방식 |
|:---|:---|
| 방문건강관리 대상자 등록 | 본인 동의 후 Soli 안부 연결 |
| 만성질환 상담 | 약 복용, 병원 방문 어려움 신호를 생활위기 체크에 반영 |
| 정신건강 상담 연계 | 외로움·절망 표현이 반복될 경우 담당자 확인 대상으로 표시 |

### 4-4. 아파트 관리사무소·공과금 상담 접점

생활위기는 행정 민원보다 먼저 공과금, 관리비, 우편물, 연락 두절에서 나타날 수 있다. 다만 이 정보는 자동 조회가 아니라 본인 동의, 기관 제휴, 또는 사용자 제출 증빙이 필요하다.

| 접점 | Solmate 연결 방식 |
|:---|:---|
| 아파트 관리사무소 | 1인가구 안부 캠페인 QR 배포, 장기 우편물 방치 시 주민센터 안내 |
| 수도·전기·가스 상담 | 체납 위험군에게 선택적 안부 연결 안내 |
| 통신비 상담 | 연락 두절 위험이 큰 고령 1인가구에 선택 안내 |
| 사용자 제출 | 고지서 사진, 문자, PDF 등 증빙을 Soli가 체크리스트로 정리 |

현장 장면:

> 어르신이 "요즘 고지서가 밀렸는데 어디 말해야 할지 몰랐다"고 말한다. Soli는 생활위기 체크리스트를 열고, 담당자는 긴급복지·감면 제도·상담 기관 후보를 함께 확인한다.

### 4-5. 병원·약국·생활지원사

병원과 약국, 생활지원사는 반복 접촉을 통해 생활 변화를 알아차릴 수 있다.

| 접점 | Solmate 연결 방식 |
|:---|:---|
| 병원 | 만성질환 시니어에게 안부 연결 포스터 안내 |
| 약국 | 복약 중단, 외출 감소가 걱정되는 주민에게 주민센터 연계 안내 |
| 생활지원사 | 방문 시 모바일 신분증 발급·가입 보조 |
| 통장·반장 | 직접 판단하지 않고 주민센터 안부 연결 신청 경로 안내 |

## 5. 서비스 이용 흐름

### 5-1. 모바일 신분증과 동의 기반 안전망의 역할

Solmate가 말해야 하는 핵심 결론은 다음과 같다.

> 모바일 신분증으로 본인확인을 완료한 사용자가 명시적으로 동의한 경우, Solmate는 위치, 생활요금 체납 징후, 공공 마이데이터 또는 사용자 제출 증빙을 결합해 고립·생활위기 신호를 확인한다. 단, 모바일 신분증은 본인확인 수단이며, 생활요금·복지 자격 자동 조회는 공공 마이데이터 및 기관 제휴가 필요한 파일럿 이후 확장 범위로 분리한다.

즉, 모바일 신분증은 모든 생활 정보를 가져오는 API가 아니다. 모바일 신분증은 "이 사람이 본인인가"를 확인하는 신뢰의 출발점이고, 위치·요금·복지 정보는 본인의 별도 동의와 기관 연계를 통해 단계적으로 붙이는 데이터 레이어다.

| 구분 | 가능한 역할 | 필요한 전제 |
|:---|:---|:---|
| 모바일 신분증 | 본인확인, 인증 결과 수신, DID/검증 상태 연결 | OmniOne CX 또는 모바일 신분증 검증 흐름 |
| 위치 | 긴급 확인·방문 연계를 위한 현재 위치 제출 | 앱 위치 권한, 명시적 동의, 목적·보유기간·철회 안내 |
| 생활요금 | 수도·전기·가스·통신비 부담 또는 체납 징후 확인 | 사용자 제출 증빙, 공공 마이데이터, 기관 제휴 |
| 복지·생활수급 정보 | 받을 가능성이 있는 복지서비스 추천과 담당자 상담 연결 | 복지서비스 API, 공공 마이데이터, 지자체 협력 |
| 담당자 지원 | 위험 신호를 보고 전화·방문·복지 연결·지원 집행 결정 | 필요한 최소 정보만 요약 제공, 담당자 권한 관리 |

이 구조가 만들어내는 사회적 의미는 아래처럼 설명한다.

> 사회취약계층이 모바일 신분증으로 본인확인을 하고, 본인이 동의한 범위 안에서 위치·생활위기 신호·복지 연결 정보를 공유하면, 담당 공무원은 기존 행정 데이터만으로는 보이지 않던 위험을 더 빨리 확인하고 직접 지원할 수 있는 또 하나의 창구를 갖게 된다.

발표용으로는 더 짧게 말한다.

> 모바일 신분증과 동의 기반 데이터 공유를 통해, 취약계층은 도움을 요청할 수 있는 안전한 통로를 얻고, 공무원은 더 빠르고 정확하게 지원할 수 있는 새로운 접점을 갖게 됩니다.

제안서 문장으로는 다음 표현을 사용한다.

> Solmate는 모바일 신분증 기반 본인확인과 동의 기반 정보 공유를 결합해, 사회취약계층이 스스로 위험 신호를 안전하게 전달하고 공무원이 직접 지원으로 연결할 수 있는 신뢰 가능한 디지털 창구를 제공한다.

표현 원칙은 "감시"가 아니라 "동의 기반 안전망"이다.

### 5-2. 55세 이상 누구나 시작 가능한 흐름

```text
[현장 접점]
주민센터 / 복지관 / 경로당 / 보건소 / 관리사무소 / 병원·약국
          ↓
[안내]
"55세 이상 안부 연결 신청" QR 또는 담당자 보조 등록
          ↓
[본인확인]
모바일 신분증 기반 OmniOne CX 인증
          ↓
[동의]
정기 안부, 현재 위치 일회성 제출, 생활위기 체크리스트, 증빙 제출 범위 선택
          ↓
[Soli 연결]
매일 또는 격일 안부 질문, 생활위기 신호 요약
          ↓
[담당자 확인]
응답 공백, 위험 표현, 생활비 어려움 언급, 위치 기반 방문 필요성 등을 우선순위로 확인
          ↓
[지원 연결]
복지서비스 추천, 긴급복지 상담, 방문 확인, 지원 집행 기록
```

### 5-3. 데이터 동의 범위

| 정보 | MVP 처리 | 파일럿 이후 확장 |
|:---|:---|:---|
| 본인확인 | 모바일 신분증 기반 인증 결과 수신 | 지자체·승인 이용기관 구조와 연계 |
| 현재 위치 | 사용자가 직접 동의한 경우 일회성 제출 | 위치기반서비스 정책 정비 후 제한적 운영 |
| 생활요금 어려움 | 체크리스트와 사용자 제출 증빙으로 확인 | 공공 마이데이터 또는 기관 제휴 API 검토 |
| 복지서비스 후보 | 한국사회보장정보원 복지서비스 API로 가능성 안내 | 개인 자격 확인은 공공 마이데이터·지자체 협력 필요 |
| 담당자 조치 | 담당자 승인·지원 연결 기록 | 지원 완료 VC/txId 기반 사후 검증 |

## 6. 극적 시나리오 초안

### 6-1. 3일 응답 공백

1. 68세 1인가구 사용자가 경로당 설명회에서 Solmate에 가입한다.
2. 매일 아침 Soli가 "오늘 식사는 하셨어요?"라고 묻는다.
3. 사용자가 며칠 전 "고지서가 밀려서 걱정"이라고 답한다.
4. 이후 3일 동안 응답이 없다.
5. 담당자 대시보드에는 "응답 공백 3일 + 최근 생활비 어려움 언급"이 상위 확인 대상으로 표시된다.
6. 담당자는 전화 확인 후 필요하면 방문 확인과 긴급복지 상담으로 연결한다.

### 6-2. 공과금 체납 상담에서 지원 연결

1. 주민센터에서 공과금 체납을 상담하던 사용자가 안부 연결 안내를 받는다.
2. 모바일 신분증으로 본인확인을 완료하고, 생활위기 체크리스트 제출에 동의한다.
3. Soli는 수도·전기·가스·통신비 부담, 식사, 병원 방문, 외출 빈도를 묻는다.
4. 담당자는 Soli 요약을 보고 긴급복지, 에너지바우처, 감면 제도 후보를 확인한다.
5. 지원 집행이 이뤄지면 VC/txId로 사후 검증 가능한 기록을 남긴다.

### 6-3. 공무원의 아침 화면

1. 담당자는 출근 후 전체 대상자를 하나씩 뒤지지 않는다.
2. 대시보드에서 "오늘 먼저 확인할 5명"을 본다.
3. 각 대상자는 응답 공백, 위기 표현, 생활비 어려움, 최근 담당자 조치 이력으로 정렬된다.
4. 담당자는 근거 요약을 보고 전화, 방문, 복지 매칭, 집행 승인 중 다음 행동을 선택한다.

## 7. 마케팅 메시지와 현장 문구

### 7-1. 포스터·QR 문구

|:---|:---|

| 복지관 | "갑자기 연락이 안 될 때, 누군가 알아차릴 수 있게." |

| 보건소 | "건강과 생활이 함께 걱정될 때, 담당자에게 신호가 닿도록." |
| 관리사무소 | "혼자 사는 이웃의 안전, 본인 동의로 시작합니다." |

### 7-2. 상담원이 설명할 한 문장

> 이 서비스는 위치나 개인정보를 몰래 보는 서비스가 아니라, 본인이 동의한 범위 안에서 안부 신호를 남겨 위급할 때 담당자가 더 빨리 확인할 수 있게 하는 안전 연결 서비스입니다.

### 7-3. 발표용 한 문장

> Solmate는 55세 이상 시니어가 주민센터와 생활 현장에서 쉽게 만나는 안부 연결 창구이며, 모바일 신분증과 동의 기반 데이터 공유로 공무원이 더 빠르고 직접적인 지원을 할 수 있게 만든다.

## 8. 제안서 반영 포인트

| 제안서 위치 | 반영할 메시지 |
|:---|:---|
| 목적 및 필요성 | 기존 복지 시스템은 신청과 행정 데이터 중심이라, 혼자 사는 시니어의 일상 변화와 응답 공백을 늦게 발견한다. |
| 상세 설명 | 주민센터·복지관·경로당·보건소에서 55세 이상 누구나 가입하고, Soli가 정기 안부와 생활위기 체크를 시작한다. |
| 차별성 | 모바일 신분증 본인확인, Soli 대화 신호, 담당자 우선순위 대시보드, 온체인 집행 기록이 하나의 현장 지원 루프로 연결된다. |
| 기대효과 | 공무원은 오늘 먼저 확인할 대상을 줄여 보고, 시니어는 도움을 요청하기 전에도 위험 신호를 안전하게 남길 수 있다. |
| 사업화 | B2G SaaS 도입 시 주민센터·복지관·보건소 캠페인을 운영 패키지로 함께 제공한다. |
