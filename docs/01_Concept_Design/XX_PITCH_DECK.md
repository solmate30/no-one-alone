## Slide 2 — Problem

**One-line message:** 시스템이 찾지 못하는 사람들이 있고, 그들은 죽어서야 발견된다.

**Content:**

| 지표 | 수치 | 출처 |
|:---|:---|:---|
| 전국 복지대상자 | 1,153만 명 | 보건복지부 |
| 복지 담당자 1인당 관리 인원 | 평균 1,027명 | 복지부 2020 |
| 발굴 후 지원 연결률 | 43% | 보건복지부 2023 |
| 위기정보 수집 주기 | 격월 (연 6회) | 복지부 위기정보시스템 |

**왜 사각지대가 생기는가:**
- 주소지 불일치: 고시원·쪽방·여관 거주자
- 자기 신청 불가: 정신질환, 인지저하, 사회적 고립
- 이웃 무관심: 1인 가구 증가로 주변 인지 자체가 없음

**기존 해결 시도의 한계:** DB손해보험 고독사 보험 — 2017년 출시, 8년간 계약 1건. 이유: 위험 임차인이 어디 있는지 알 수 없어서.

---

## Slide 3 — Solution

**One-line message:** 국가가 못 찾는 사람을, 이웃이 찾고, AI가 연결하고, 블록체인이 증명한다.

**Content:**

```
[발굴]  이웃 제보 / 가족 케어 등록 / 자가 신청
   ↓
[판단]  Soli AI 위기도 스코어링 + 모바일 신분증 신원 확인
   ↓
[집행]  담당자 대시보드 → 승인 → 지원 연결
   ↓
[기록]  OmniOne Chain 온체인 투명 기록
```

핵심 차별점: 신청을 기다리지 않는다. Soli가 먼저 말을 건다.

**Content:**

|:---|:---|

| SOM | 파일럿 10개 동 + 보험사 1개 (2027 목표) |

**성장 트렌드:**

- 한국 1인 가구 비율 지속 증가
- K-복지 플랫폼 수출 가능성 (일본·대만)

## Slide 5 — Product

**One-line message:** MVP는 이미 작동한다 — Soli가 지금 대화 중이다.

**Content:**

**핵심 유저 플로우 (3단계):**

1. 이웃이 Solmate 앱에서 이상 징후 제보 → 모바일 신분증으로 본인 확인
2. Soli가 대상자에게 접촉 → 대화 분석 → 위기도 스코어 산출
3. 담당자 대시보드에 우선순위 표시 → 승인 → OmniOne Chain 기록

**현재 구현 완료:**
- Next.js + TypeScript 서비스 배포
- Claude API 기반 Soli 챗봇
- 한국 공공데이터 API 연동
- 담당자 대시보드 UI

**해커톤 결선 추가 예정:** 모바일 신분증(OmniOne CX) + Open DID + OmniOne Chain 연동

## Slide 6 — Business Model

**One-line message:** 복지 문제를 해결하면서 4개 레이어 수익이 발생한다.

**Content:**

| 레이어 | 수익 모델 | 규모 (추정) |
|:---|:---|:---|
| B2G SaaS | 담당자 1인 월 10만원 구독 | 전국 3,500개 동 → 월 3.5억 |
| B2B2C 보험 | 보험사 위기 감지 솔루션 이용료 | 보험사 영업망으로 전국 자동 확산 |
| B2B CSR | 집행액 3~5% 수수료 + ESG 리포트 월 50만원 | 기업 CSR 예산 투명화 수요 |
| B2B 데이터 API | 보험사 언더라이팅 API 구독 | 10개 손보사 → 연 10억 |

**왜 보험사가 원하는가:**
- 기존 고독사 보험: 리스크 데이터 없어 판매 불가
- Solmate 탑재 후: 실시간 모니터링 → 선제 개입 → 보험금 지출 감소 + 상품 경쟁력 확보

## Slide 8 — Competition

**One-line message:** 기존 시스템은 찾을 수 없고, 우리는 먼저 다가간다 — 그리고 데이터가 쌓일수록 격차가 벌어진다.

**Content:**

| 구분 | 기존 복지 시스템 | 고독사 보험 | Solmate |
|:---|:---|:---|:---|
| 발굴 방식 | 공무원 수동, 격월 | 없음 | AI 상시 + 시민 제보 |
| 사각지대 | 데이터 없으면 못 찾음 | 데이터 없음 | 이웃 제보로 커버 |
| 신청 방식 | 본인 신청 필수 | 해당 없음 | AI가 먼저 접촉 |
| 집행 투명성 | 추적 불가 | 해당 없음 | 온체인 전 과정 기록 |
| 데이터 축적 | 없음 | 없음 | 사회적 고립 데이터셋 누적 |

**핵심 해자 (Moat):** 서비스 사용이 늘수록 쌓이는 사회적 고립 데이터 — 기존 어디에도 없는 데이터셋. 이 데이터가 보험사 언더라이팅을 가능하게 하고, 경쟁자 진입 장벽이 된다.
