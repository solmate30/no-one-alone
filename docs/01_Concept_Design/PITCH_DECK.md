## 3. 제안내용의 목적 및 필요성

**One-line message**: 시스템이 찾지 못하는 사람들이 있고, 그들은 죽어서야 발견된다.

### 핵심 메시지

복지 사각지대의 문제는 지원 제도가 없어서가 아니라, 위기 가구를 제때 찾지 못해서 발생한다.

### 문제 현황

| 지표 | 수치 | 의미 |
|:---|:---:|:---|
| 전국 복지대상자 | 1,488.5만 명(2023) | 2019년 1,153.1만 명 대비 335.4만 명 증가 |
| 복지 담당자 관리 부담 | 1인 평균 1,027명(2020 보도 기준) | 수동 발굴에 구조적 한계가 있음 |
| 발굴 후 실제 지원 | 누적 666만 명 중 290만 명(약 44%) | 발굴 이후에도 절반 이상이 지원으로 연결되지 않음 |
| 위기정보 수집 주기 | 격월(연 6회) | 상시 위기 감지가 어려움 |

출처: 보건복지부 2024 복지 사각지대 발굴 보도자료, 보건복지부 제출 통계 기반 보도(뉴시스, 2024), 한국일보 2020 보도

### 사각지대가 생기는 이유

- 주소지와 실거주지가 다른 고시원·쪽방·여관 거주자는 행정 데이터에 잘 잡히지 않는다.
- 정신질환, 인지저하, 사회적 고립 상태의 대상자는 본인이 직접 신청하기 어렵다.
- 1인 가구 증가로 주변 이웃이 이상 징후를 알아차려도 신고 채널이 명확하지 않다.
- 기존 고독사 보험은 위험 대상자를 알 수 없어 판매와 리스크 관리 모두 어렵다.

## 4. 제안내용의 상세 설명

**One-line message**: 국가가 못 찾는 사람을, 이웃이 찾고, AI가 연결하고, 블록체인이 증명한다.

### 서비스 정의

no-one-alone은 "발굴 → 판단 → 집행 → 기록"을 하나로 연결하는 복지 위기 대응 플랫폼이다.

```text
[발굴] 이웃 제보 / 가족 케어 등록 / 자가 신청
  -> [판단] Soli AI 위기도 스코어링 + 모바일 신분증 확인
  -> [집행] 담당자 대시보드 우선순위 확인 + 지원 승인
  -> [기록] OmniOne Chain에 집행 증명 해시 기록
```

### 핵심 기능

| 기능 | 대상 | 설명 |
|:---|:---|:---|
| 이웃 제보 | 시민 | 주소와 상황을 입력하고 모바일 신분증으로 제보자 신원을 확인한다. |
| 가족 케어 등록 | 3040 자녀 | 떨어져 사는 부모님에게 Soli가 주기적으로 말을 걸고 이상 패턴을 감지한다. |
| 자가 복지 신청 | 위기 대상자 | Soli와 대화하며 받을 수 있는 복지를 추천받고 자격 확인으로 신청을 연결한다. |
| Soli AI | 대상자·가족 | 감정 표현, 위기 키워드, 응답 공백을 기반으로 위기도를 산출한다. |
| 담당자 대시보드 | 복지 담당자 | 오늘 우선 케어가 필요한 사람을 스코어 순으로 확인한다. |
| 온체인 기록 | 지자체·기업 | 제보·판단·집행의 증명 해시를 기록해 투명성을 높인다. |

### 모바일 신분증 활용 지점

| 사용 시점 | 역할 | 효과 |
|:---|:---|:---|
| 이웃 제보 시 | 제보자 본인 확인 | 허위 신고 방지 |
| 복지 자격 확인 시 | 대상자 본인 확인 | 자격 검증 간소화 |
| 지원 집행 직전 | 수급자 본인 확인 | 부정 수급 방지 |

## 5. 제안내용의 차별성

**One-line message**: 기존 시스템은 찾을 수 없고, no-one-alone은 먼저 다가간다. 데이터가 쌓일수록 격차가 벌어진다.

### 기존 시스템과의 차이

| 구분 | 기존 복지 시스템 | 고독사 보험 | no-one-alone |
|:---|:---|:---|:---|
| 발굴 방식 | 공무원 수동, 격월 | 없음 | AI 상시 + 시민 제보 |
| 사각지대 | 데이터 없으면 못 찾음 | 데이터 없음 | 이웃 제보로 커버 |
| 신청 방식 | 본인 신청 필수 | 해당 없음 | AI가 먼저 접촉 |
| 집행 투명성 | 추적 불가 | 해당 없음 | 온체인 전 과정 기록 |
| 담당자 업무 | 다수 대상 수동 판단 | 해당 없음 | 위기도 우선순위 기반 판단 |
| 데이터 축적 | 없음 | 없음 | 사회적 고립 데이터셋 누적 |

### 기술 차별성

- 모바일 신분증을 억지로 붙이는 것이 아니라, 제보·자격·집행 신뢰 구조의 핵심으로 사용한다.
- Open DID는 제보자 신원 증명에 활용한다. 결선 MVP에서 제보자 인증 1개 플로우로 연동 범위를 제한한다.
- OmniOne Chain은 개인정보 원문이 아니라 DID 해시, 데이터 해시, 집행 증명 해시만 기록한다.
- Soli AI는 단순 상담 챗봇이 아니라 복지 담당자에게 전달할 위기 신호를 정리하는 첫 접점이다.

**핵심 해자 (Moat)**: 서비스 사용이 늘수록 기존 공공 데이터로는 포착하기 어려운 사회적 고립 신호 데이터셋이 축적된다. 이 데이터는 향후 보험사 리스크 모델과 지자체 선제 대응 정책의 기반이 되어 경쟁자 진입 장벽이 된다.

## 6. 결선 MVP 개발·시연 계획

**One-line message**: 결선 MVP는 명확하다. Soli가 먼저 감지하고 담당자가 바로 움직인다.

### 현재 상태

현재 구현된 웹앱, API, DB, OmniOne 연동은 없다. 5월 제안서 제출 후 예선에 선정되면, 결선 기간(7/1~9/21) 동안 아래 MVP를 개발한다.

### 결선 MVP 포함 범위

| 범위 | 구현 목표 | 시연 장면 |
|:---|:---|:---|
| Soli 챗봇 | 위기 키워드 감지와 위기도 스코어 출력 | 사용자가 위험 문장을 입력하면 스코어가 상승 |
| 이웃 제보 | 주소·상황 입력과 제보자 인증 | 모바일 신분증 인증 후 제보 접수 |
| 담당자 대시보드 | 위기 우선순위 리스트와 상세 요약 | 담당자가 고위험 케이스를 확인 |
| 지원 승인 | 지원 종류 선택과 승인 상태 변경 | 담당자가 지원 집행을 승인 |
| OmniOne Chain | 증명 해시 기록과 TX 해시 표시 | 집행 이벤트의 TX 해시 확인 |

### 제외 범위

- 실제 행정 시스템 연동
- 실제 보험사 API 계약·연동
- 실제 복지금 지급·정산
- 실사용자 개인정보 수집 (데모 데이터 사용)

### 사회적 효과

- 고독사 발생 전 선제 개입 가능성을 높인다.
- 시민 참여를 통해 기존 행정 데이터가 포착하지 못하는 사각지대를 보완한다.
- 온체인 증명으로 복지 집행과 CSR 기부의 투명성을 강화한다.

## 8. 사업화 계획

**One-line message**: B2G SaaS로 시작해서, 보험과 CSR로 확장한다.

### 1차 수익모델: B2G SaaS

행정복지센터 담당자용 AI 우선순위화 도구를 월 구독으로 공급한다.

| 단계 | 규모 | 월 매출 |
|:---|:---|:---|
| 파일럿 | 10개 동 | 100만원 |
| 확장 | 100개 동 | 1,000만원 |
| 전국 | 3,500개 동 | 3.5억 |

진입 전략: 해커톤 입상 후 파일럿 지자체 MOU를 추진하고, 공공 PoC 성과를 기반으로 조달·협약 채널을 검토한다.

### 확장 경로

| 레이어 | 수익 모델 | 진입 조건 |
|:---|:---|:---|
| B2B2C 보험 | 보험사 위기 감지 솔루션 이용료 | B2G 파일럿 데이터로 리스크 모델 검증 후 |
| B2B CSR | 집행액 3~5% 수수료 + ESG 리포트 구독 | 온체인 집행 기록이 기업 ESG 보고에 활용될 수 있음을 확인 후 |
| 데이터 API | 익명 집계 데이터 API 구독 | 개인 단위 판매 없음. 집계·비식별 처리된 지역·연령대 통계만 제공. 결선 MVP 제외 범위 |

**보험사가 필요한 이유**: 기존 고독사 보험은 위험군 식별과 리스크 산정 데이터가 부족해 상품 확산이 제한적이다. no-one-alone이 축적하는 사회적 고립 스코어는 보험사가 리스크를 계산하고 상품을 설계할 수 있게 한다.

**개인정보 원칙**: 보험·CSR·데이터 API 수익 모두 집계·비식별 처리된 데이터만 활용한다. 개인 단위 데이터 판매는 없다.

### 성장 로드맵

| 단계 | 기간 | 목표 |
|:---|:---|:---|
| 생존 | 2026 | 해커톤 입상 + 결선 MVP 구현 |
| 검증 | 2027 | 파일럿 지자체 10개 + 보험사 1개 PoC |
| 성장 | 2028 | 보험사 정식 계약 + CSR 기업 확대 |
| 확장 | 2029+ | 일본·대만 고령화 시장 진출 |
