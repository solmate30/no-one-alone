## 1. Problem (문제)

**Top 3 Pain Points:**

1. 복지대상자 1,488.5만 명(2023) — 2019년 대비 335.4만 명 증가. 같은 기간 복지대상자는 약 29% 늘었지만 사회복지 담당 공무원은 약 10% 증가해 현장 업무 부담이 커졌다.
2. 시스템이 포착하지 못하는 계층이 세 가지 구조적 이유로 발생한다:
   - **신청주의 장벽**: 정신질환·발달장애 대상자는 복잡한 서류 증빙을 스스로 준비하기 어렵다
   - **수급자 배제 역설**: 소액 수급 중이라는 이유로 위기 감지 명단에서 제외되는 Proxy Trap — 이미 도움을 받고 있다고 간주된 순간 더 심각한 위기가 시스템 밖으로 빠진다
   - **아동복지 단절 절벽**: 시설 보호 아동의 만 18세 퇴소 후 위기 가정 복귀 구조
3. 복지 사각지대 발굴 후 실제 지원 약 43.6% — 누적 발굴 666만 명 중 290만 명 지원. 비수급 빈곤층 66만 명(2021)처럼 제도 요건에 가까운 어려움을 겪지만 실제 지원에서 배제되는 사각지대가 존재한다.

**기존 대안의 한계:**

| 대안 | 한계 |
|:---|:---|
| 복지부 위기정보시스템 | 격월(연 6회) 데이터 수집, 상시 감지 불가 |
| 위기가구 발굴 명단 | 이미 수급 중인 가구를 발굴 대상에서 제외 — Proxy Trap으로 실제 위기 가구 이탈 |
| 고독사 보험 | 임대인의 특수청소·유품 정리·원상복구·임대료 손실을 보상하는 초기 시장이 형성되고 있으나, 사고 빈도와 지역별 위험률 데이터가 부족해 보험료 산정과 상품 확산에 한계 |
| 공무원 수동 조사 | 인력 부족, 자기 신청 불가 계층 미발굴 |

## 2. Customer Segments (고객 세그먼트)

**얼리 어답터:**
- B2G: 고독사 관리 파일럿 의지가 있는 기초지자체 행정복지센터
- B2B2C: 고독사 보험의 보험료 산정·위험률 분석 데이터가 필요한 손해보험사
- B2C: 복지 정보나 긴급 도움을 직접 찾는 취약계층. 보호자 접점은 파일럿 이후 검토

## 3. Unique Value Proposition (UVP)

> "먼저 말을 거는 국가 — AI가 24/7 감지하고, 복지가 닿고, 블록체인이 증명한다"

- 신청을 기다리지 않고 Soli가 먼저 접촉
- Soli 정기 안부 + AI 스코어링으로 데이터 없는 사각지대까지 커버
- 모바일 신분증(OmniOne CX)으로 가입·로그인과 집행 직전 본인 확인

## 4. Solution (솔루션)

```
[발굴 — Soli 1차 감지]
담당자 등록 기반 정기 안부 / Soli 복지 매칭
          ↓
[판단 — AI 플랫폼]
위기도 스코어링 + Soli 챗봇 케어
모바일 신분증으로 신원 확인
          ↓
[집행 — B2G 담당자 도구]
행정복지센터 대시보드 → 승인 → 지원 집행
          ↓
[기록 — OmniOne Chain]
지원 확인 VC 발급-판단-지원 완료 증명 해시 기록
```

## 5. Channels (채널)

| 채널 | 대상 | 방식 |
|:---|:---|:---|
| 정부 SW 조달 등록 | B2G | 수의계약 진입 |
| 보험사 리스크 데이터 PoC | B2B2C | 손해보험사에 권역·연령대·가구 유형별 비식별 고립 신호 리포트 제공 |
| 해커톤 입상 + 창업 지원 | 초기 | 공신력 확보, 파일럿 연결 |
| 공공 API 생태계 | B2G | 복지 담당자 직접 도달 |

## 6. Revenue Streams (수익 구조)

| 레이어 | 모델 | 규모 (추정) |
|:---|:---|:---|
| Layer 1 (주력, 2026~) — B2G SaaS | 기초지자체 단위 연 2,000만~3,000만원 (담당자 40명 기준 월 4~5만원/인 수준) | 3년 목표: 10개 기초지자체 → 연 2.5억 / 전국 TAM: 226개 → 연 56억 |
| Layer 2 (파일럿 후, 2027~) — 보험사 리스크 데이터 PoC | 비식별 리스크 리포트 구독 또는 공동 연구 | 파일럿 성과 확보 후 손보사 1개 PoC → 고독사 보험 보험료 산정·위험 지역 분석 참고 지표 제공 |
| Layer 3 (장기, 2029+) — 데이터 API | 익명 집계 리스크 데이터 구독 | 수천 명 이상 축적 후 k-익명성 등 재식별 방지 기준 충족 범위에서 장기 검토 |

## 7. Cost Structure (비용 구조)

- 인건비 (개발 2인 기준): 초기 핵심 비용
- 인프라 (Vercel + AWS): 서비스 운영
- OmniOne CX 연동 및 블록체인 트랜잭션 비용
- 파일럿 영업 및 규제 대응

## 8. Key Metrics (핵심 지표)

| 지표 | 현재 | 목표 (3년) |
|:---|:---|:---|
| 발굴 후 지원 연결률 | 약 43.6% | 70% 이상 |
| 위기정보 수집 주기 | 격월 | 상시 |
| 지원 처리 기간 | 수주 | 24시간 이내 |
| 담당자 실질 부담 | 복지대상자 증가율 29% vs 담당 공무원 증가율 10% | 케이스당 확인 소요 시간 50% 감소 |
| 제도 밖 사각지대 | 비수급 빈곤층 66만 명 | 파일럿 지역 내 미연결 위험 신호 지속 감소 |

## 9. Unfair Advantage (데이터 경쟁 우위)

**데이터 우위**: no-one-alone이 축적하는 사회적 고립 신호 데이터 (고립도 스코어, 응답 공백, Soli 감정 패턴, 담당자 처리 결과) — 기존 공공 데이터로는 포착하기 어려운 위험 신호를 보완하는 데이터셋

**네트워크 효과**: Soli 접점과 담당자 피드백이 쌓일수록 위기 신호 분류 품질 향상 → 지자체 파일럿 성과 확보 → 보험사 파트너 확장
