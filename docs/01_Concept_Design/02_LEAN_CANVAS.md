# Lean Canvas — no-one-alone
> Created: 2026-05-08 01:39
> Last Updated: 2026-05-13 12:30
> Backlog: P-02, P-03, P-06, D-02

## 1. Problem (문제)

**Top 3 Pain Points:**

1. 복지대상자 1,488.5만 명(2023) — 2019년 대비 335.4만 명 증가, 현장 담당자 부담 가중
2. 빅데이터망 바깥의 계층 (고시원·쪽방 거주자, 자기 신청 불가 대상자) 은 시스템에 포착되지 않음
3. 복지 사각지대 발굴 후 실제 지원 약 44% — 누적 발굴 666만 명 중 290만 명 지원

**기존 대안의 한계:**

| 대안 | 한계 |
|:---|:---|
| 복지부 위기정보시스템 | 격월(연 6회) 데이터 수집, 상시 감지 불가 |
| 고독사 보험 (DB손보 등) | 위험군 식별·리스크 산정 데이터 부족으로 상품 확산 한계 |
| 공무원 수동 조사 | 인력 부족, 자기 신청 불가 계층 미발굴 |

## 2. Customer Segments (고객 세그먼트)

**얼리 어답터:**
- B2G: 고독사 관리 파일럿 의지가 있는 기초지자체 행정복지센터
- B2B2C: 신규 고독사 보험 상품 출시를 검토 중인 손해보험사
- B2C: 복지 정보나 긴급 도움을 직접 찾는 취약계층. 보호자 접점은 파일럿 이후 검토

## 3. Unique Value Proposition (UVP)

> "먼저 말을 거는 국가 — AI가 24/7 감지하고, 복지가 닿고, 블록체인이 증명한다"

- 신청을 기다리지 않고 Soli가 먼저 접촉
- Soli 정기 안부 + AI 스코어링으로 데이터 없는 사각지대까지 커버
- 모바일 신분증(OmniOne CX)으로 가입·로그인과 집행 직전 본인 확인

## 4. Solution (솔루션)

```
[발굴 — Soli 1차 감지]
담당자 등록 기반 정기 안부 / Soli 복지 매칭
          ↓
[판단 — AI 플랫폼]
위기도 스코어링 + Soli 챗봇 케어
모바일 신분증으로 신원 확인
          ↓
[집행 — B2G 담당자 도구]
행정복지센터 대시보드 → 승인 → 지원 집행
          ↓
[기록 — OmniOne Chain]
지원 확인 VC 발급-판단-지원 완료 증명 해시 기록
```

## 5. Channels (채널)

| 채널 | 대상 | 방식 |
|:---|:---|:---|
| 정부 SW 조달 등록 | B2G | 수의계약 진입 |
| 보험사 공동 실증 | B2B2C | 보험사를 통해 고령층 예방형 서비스 파트너십 확산 |
| 해커톤 입상 + 창업 지원 | 초기 | 공신력 확보, 파일럿 연결 |
| 공공 API 생태계 | B2G | 복지 담당자 직접 도달 |

## 6. Revenue Streams (수익 구조)

| 레이어 | 모델 | 규모 (추정) |
|:---|:---|:---|
| Layer 1 (주력, 2026~) — B2G SaaS | 기초지자체 단위 연 2,000만~3,000만원 (담당자 40명 기준 월 4~5만원/인 수준) | 3년 목표: 10개 기초지자체 → 연 2.5억 / 전국 TAM: 226개 → 연 56억 |
| Layer 2 (파일럿 후, 2027~) — 보험사 공동 실증 | 공동 실증 용역비 | 파일럿 성과 확보 후 손보사 1개 PoC → 정식 계약 |
| Layer 3 (장기, 2029+) — CSR·데이터 API | 집행액 수수료 + 비식별 데이터 구독 | 온체인 기록 및 대규모 데이터 축적 후 장기 검토 |

## 7. Cost Structure (비용 구조)

- 인건비 (개발 2인 기준): 초기 핵심 비용
- 인프라 (Vercel + AWS): 서비스 운영
- OmniOne CX 연동 및 블록체인 트랜잭션 비용
- 파일럿 영업 및 규제 대응

## 8. Key Metrics (핵심 지표)

| 지표 | 현재 | 목표 (3년) |
|:---|:---|:---|
| 발굴 후 지원 연결률 | 약 44% | 70% 이상 |
| 위기정보 수집 주기 | 격월 | 상시 |
| 지원 처리 기간 | 수주 | 24시간 이내 |
| 담당자 실질 부담 | 고부담 | 50% 감소 |

## 9. Unfair Advantage (핵심 해자)

**데이터 Moat**: no-one-alone이 축적하는 사회적 고립 신호 데이터 (고립도 스코어, 응답 공백, Soli 감정 패턴, 담당자 처리 결과) — 기존 공공 데이터로는 포착하기 어려운 위험 신호를 보완하는 데이터셋

**네트워크 효과**: Soli 접점과 담당자 피드백이 쌓일수록 위기 신호 분류 품질 향상 → 지자체 파일럿 성과 확보 → 보험사·CSR 파트너 확장
